# Example Repository Structure (Canonical Appendix Demo)

```text
paper/
├── sections/
├── figures/
├── publication/
├── specs/
├── workflows/
└── audits/
```

## Mapping to Demonstration Artifacts
- `specs/`: synchronization, workflow, publication, and visual synapse specs.
- `workflows/`: recursive issue loop and checkpoint manifest.
- `audits/`: reflective audit evidence.
- `publication/`: manifest + renderer declarations (`00README.json`).
- `diagrams/`: synchronization loop and rendering pipeline diagrams.
