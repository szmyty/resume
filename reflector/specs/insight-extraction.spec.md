# Purpose

This specification defines a reusable process for extracting:
- scoped insights
- conceptual breakthroughs
- recurring patterns
- emotional realizations
- systems-level ideas
- philosophical concepts
- technical observations
- reflective learnings

from:
- conversations
- PDFs
- journals
- repositories
- transcripts
- research papers
- notes
- therapy discussions
- long-form chat histories

The output should create:

> a reusable catalog of scoped conceptual artifacts.

---

# Core Philosophy

The goal is NOT:
- summarizing everything
- preserving every detail
- generating transcripts
- exhaustive archival

The goal IS:
- identifying meaningful insight clusters
- compressing high-signal concepts
- extracting reusable ideas
- preserving conceptual breakthroughs
- externalizing meaningful cognition

This process acts like:

> reflective signal extraction from dense information.

---

# Input Sources

The source material may include:
- long chat histories
- therapy conversations
- emotional reflections
- brainstorming sessions
- technical discussions
- research PDFs
- philosophical writing
- repositories
- meeting notes
- transcripts
- journals

The system should:
- interpret the source holistically
- identify recurring themes
- detect conceptual clusters
- extract high-signal ideas

---

# Extraction Goals

The extraction process should prioritize:
- meaningful realizations
- recurring patterns
- systems-level observations
- emotionally important breakthroughs
- philosophical insights
- technical innovations
- workflow discoveries
- reflective observations
- conceptual correlations

Avoid:
- trivial details
- conversational filler
- redundant phrasing
- low-signal content
- emotionally unsafe overexposure

---

# Scoped Insight Philosophy

Each extracted insight should represent:

> one coherent conceptual cluster.

An insight should ideally:
- stand on its own
- be understandable independently
- contain one primary idea
- preserve conceptual coherence
- remain compressible into visual form later

Avoid:
- giant mixed-topic extractions
- excessively broad categories
- unrelated idea merging

---

# Conversation-Aware Behavior

If the system has access to:
- long-term memory
- chat history
- conversation context
- prior discussions

it should:
- identify recurring themes across time
- detect evolution of ideas
- identify conceptual progression
- preserve emotional/contextual continuity where useful

This is especially valuable for:
- therapy reflections
- personal growth
- recursive systems thinking
- long-term projects
- philosophy development

---

# Output Structure

The extraction output should be:
- lightweight
- readable
- iterable
- reusable
- easy to copy/paste
- easy to manually curate

Preferred formats:
- markdown
- structured bullet lists
- lightweight JSON
- grouped conceptual sections

Avoid:
- deeply nested complex schemas
- overengineered metadata
- rigid database-style formatting

---

# Preferred Markdown Format

Recommended structure:

    # Insight Catalog

    ## Insight 001

    Title:
    Summary:
    Key Concepts:
    Why It Matters:
    Possible Visual Directions:
    Tags:

    ---

    ## Insight 002

    ...

The structure should optimize for:
- readability
- skimming
- manual iteration

---

# Batch Extraction Behavior

Large sources should be processed incrementally.

Preferred behavior:
- extract in scoped batches
- avoid overwhelming outputs
- preserve conceptual clarity

Possible strategies:
- batches of 5
- batches of 10
- thematic grouping
- chronology grouping
- conceptual clustering

The process should support:

> progressive reflective extraction over time.

---

# Emotional Safety Rules

Especially for therapy/personal conversations:

The system should:
- generalize where appropriate
- avoid unnecessary specificity
- avoid exposing sensitive identities
- preserve emotional meaning without raw oversharing

The goal is:
- reflective understanding
NOT:
- emotional data dumping
